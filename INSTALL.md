# Инструкция по установке и запуску

## Предварительные требования

1. Node.js (версия 16 или выше)
2. npm или yarn
3. Expo CLI (устанавливается автоматически через npx)
4. Для запуска на устройстве:
   - iOS: iPhone с приложением Expo Go (App Store)
   - Android: Android устройство с приложением Expo Go (Google Play)

## Установка

1. Перейдите в директорию проекта:
```bash
cd "/Users/pskripilev/Downloads/Мои файлы/healt"
```

2. Установите зависимости:
```bash
npm install
```

## Запуск в режиме разработки

1. Запустите Expo сервер:
```bash
npm start
# или
npx expo start
```

2. Выберите способ запуска:
   - Нажмите `a` для Android эмулятора
   - Нажмите `i` для iOS симулятора
   - Отсканируйте QR-код в приложении Expo Go на вашем устройстве

## Первый запуск приложения

1. После открытия приложения перейдите в "Настройки"
2. Нажмите кнопку "Сгенерировать начальные данные (7 дней)"
3. Включите "Включить симуляцию" для автоматической генерации данных
4. Выберите профиль пользователя (по умолчанию "Активный")

## Сборка APK

### Установка EAS CLI

```bash
npm install -g eas-cli
```

### Настройка проекта

1. Войдите в аккаунт Expo:
```bash
eas login
```

2. Настройте проект:
```bash
eas build:configure
```

3. Создайте build:
```bash
eas build --platform android --profile preview
```

Для production build:
```bash
eas build --platform android --profile production
```

## Устранение проблем

### Ошибка при установке зависимостей

Если возникают проблемы с зависимостями:

```bash
rm -rf node_modules package-lock.json
npm install
```

### Ошибки с нативной компиляцией

Для Expo Go большинство нативных модулей уже включены. Если используете кастомную сборку, убедитесь, что все нативные модули правильно настроены.

### Проблемы с базой данных

База данных SQLite создается автоматически при первом запуске. Если возникают проблемы, приложение пересоздаст базу данных.

## Структура данных

Приложение использует SQLite для локального хранения данных:

- `health_metrics` - все метрики здоровья
- `goals` - цели пользователя
- `recommendations` - рекомендации системы

Все данные хранятся локально на устройстве.
